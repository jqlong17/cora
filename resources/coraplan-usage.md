# CoraPlan 使用说明

## 模块目的

CoraPlan 采用**更有约束的方法论**来规范 AI 写 Plan 与执行，从而：

* **减少幻觉**：通过 DDD、TDD 等约定与「完成标准 / 验收」前置，约束 AI 输出更可控、可验收的计划。

* **减少人类验收频次**：要求测试与功能同步开发，每个模块须有单元测试与 e2e 测试，**仅当测试全部通过后**才能将任务标记为完成，从而用自动化测试降低人工逐项验收的成本。

推荐在 **Agent 模式**下直接让 AI 根据约束文档生成 plan 并执行，避免使用 Cursor 自带的 Plan 功能以保持与约束一致。

***

## 如何使用

1. **首次使用**：在 CoraPlan 视图为空时，点击「在此工作区注入 CoraPlan」，将约束文档与模板写入当前工作区的 `.cursor/plans` 与 `.cursor/rules`。
2. **打开约束说明**：点击视图右上角「打开 Plan 约束说明」，阅读 `00-PLAN-CONSTRAINTS.md`，了解如何写 plan、测试要求与禁止项。
3. **打开 README 登记表**：点击「打开 README 登记表」，查看或维护 plan 文件登记表（编号、文件名、简介）。
4. **写 Plan**：在 Agent 模式下让 AI 根据约束生成 `NN-名称.plan.md`，并在同一会话中按 plan 执行；或在本目录下手动新建 plan 文件并遵守约束中的编号与结构要求。
5. **管理 Plan**：在视图中点击 plan 文件可打开编辑；右键可删除、复制路径、在 Finder 中打开等。

***

## 如何配置

CoraPlan 的能力主要由**约束文档**定义，扩展本身无需单独配置即可使用。可选相关设置：

* **界面语言**：在 **设置 → Cora** 中可设置「界面语言」（默认中文），影响空态介绍与部分文案。

约束与规则在「注入」后位于工作区 `.cursor/plans/` 与 `.cursor/rules/`，可直接编辑以适配团队习惯；再次注入会覆盖约束与规则文件，README 不会覆盖已有内容。
